# AgentBridge 后续增强事项

> 状态：Backlog
> 更新日期：2026-07-13
> 用途：保存完整生产化阶段需要考虑的事项，不纳入首期 PoC 的开发范围。

## 1. 使用原则

首期目标是用若干遗留系统快速验证“智能体能否通过统一能力、CLI、中心端分用户会话和可信认证卡片可靠完成真实任务”。本文件中的事项默认暂缓，不因暂缓而丢失；当 PoC 证明路线可行并准备开放生产写入时，再按风险逐项纳入。最终用户设备零连接器、秘密绕过模型、不同用户会话隔离属于基线决策，不在本 Backlog 中延期。

PoC 仍须保留以下最低安全底线：

- 不同用户使用中心端不同 HTTP Session 和浏览器 Profile/Context，不共享 Cookie；
- 最终用户设备不安装 Chrome 扩展、本地 Daemon 或遗留系统连接器；
- 账号密码、Cookie 和 Token 不进入模型上下文和普通日志；
- 普通表单登录通过服务端生成的可信认证卡片和独立秘密通道进入中心凭据代理；
- 智能体只能调用白名单业务能力，不能获得任意 Shell、HTTP、JavaScript 或选择器；
- 不执行付款、删除、正式归档等高风险动作；
- 所有安全写入都有人工确认、结果回读和持久化操作记录；
- 失败或结果不确定时停止，不盲目重试。

## 2. 生产写入前必须补齐

### 2.1 模型数据面和云端智能体信任

- 定义 `ModelDataPolicy`：数据分级、允许的模型/供应商/区域、可进入提示和日志的字段、训练和保留策略、附件规则；
- 内网先完成结构化提取、字段裁剪和脱敏，只把必要数据交给模型；
- 个人或未知云端智能体默认不能访问敏感能力；
- 建立模型出口 DLP、数据驻留、第三方处理者和合同清单。

### 2.2 双轴风险与跨能力数据流

- 风险由 `EffectRisk × DataSensitivity × ExportVolume` 共同决定；
- 工资、病历、客户清单等高敏只读不能按普通 R0 自动放行；
- 数据分类随工作流传播，限制敏感数据被写入邮件、备注、附件或其他系统；
- 高敏读取、批量导出和跨域发送需要限量、step-up 或人工授权。

### 2.3 复杂身份、代办和人员生命周期

- 支持一个用户在同一系统中的多个合法账号/角色上下文；
- 增加 `DelegationGrant`，表达代办人、被代表人、范围、期限、理由和授权来源；
- 建立 IdentityBinding 注册证明、防错绑和定期复核；
- 完成 Joiner/Mover/Leaver 状态机，处理调岗、离职、重新入职及在途操作；
- 机器任务和 service principal 不得替代本应逐用户执行的写操作。

### 2.4 可信认证/字段/授权卡片与凭据代理生产化

当前单机 Windows PoC 已实现 `AuthChallenge`、固定登录字段 Schema、Origin/Host/CSRF 校验、页面指纹、nonce、TTL、一次性消费、内存凭据、真实页面登录、下游身份核验和 DPAPI 会话保存；还实现了业务 `FieldSubmission` 与执行授权的独立卡片、Schema 哈希、用户/会话/能力版本绑定和一次性状态机。Seeyon OA 已完成“认证卡 → 字段卡 → 授权卡 → 保存待发 → 服务器回读”的单用户真实验证。以下仍属于生产化增强，不因 PoC 成功而省略：

- 将本地协议升级为带调用方身份认证、宿主证明、设备绑定、限流、撤销和跨用户重放测试的正式协议；当前非回环服务默认强制 TLS，仅为受控内网 PoC 提供显式固定私网 IP HTTP 例外，尚未验证生产证书、反向代理信任链和真实手机网络；
- 认证卡片与模型聊天视图隔离，密码管理器、系统自动填充和生物识别解锁只服务于可信宿主，秘密不得进入模型、MCP、推送、截图、HAR、剪贴板和分析埋点；
- 字段卡片与执行授权卡片必须保持独立 Origin/页面语义和清晰系统标识，防止把“填写业务内容”伪装成“批准副作用”；生产环境为暂存业务字段和冻结计划接入 Vault/KMS 信封加密、字段级保留/删除策略、DLP 和访问审计，不能长期依赖 SQLite 明文；
- 文件上传型字段只接受用户绑定、短期、一次性的受控对象句柄，完成恶意内容扫描、类型/大小校验和目标系统回读后再消费；不得把附件内容或本地路径放入模型工具参数；
- 凭据代理使用独立服务身份和最小权限；默认秘密仅驻留内存，持久保存必须取得用户明确同意并接入 Vault/KMS、轮换、撤销、访问审计和删除流程；
- 固定 Broker、会话状态存储和能力 Worker 的密钥访问边界：Windows DPAPI 模式下三者必须运行在同一受限服务安全主体中；开发环境已验证跨 Windows 用户解密会失败关闭。多主机、容器和滚动发布不得依赖用户级 DPAPI，应迁移到带工作负载身份认证的 Vault/KMS 信封加密；
- 建立密码错误、账号锁定、验证码刷新、短信/TOTP、MFA 推送、挑战超时、设备撤销和异常频率限制；
- 登录成功后从可信页面状态或后端接口核验实际遗留账号，身份不匹配时隔离会话并告警；
- 使用第二个真实遗留账号和独立 Worker OS 身份完成会话、Profile、下载、截图和日志的不可交叉读取验证；当前仅有单用户实证；
- 为滑块、二维码同机扫码、USB Key、ActiveX、客户端证书和电子签章定义受控桌面接管或不支持策略，远程浏览器不得成为手机默认交互。
- 将一期 `InteractionEnvelope + poll/resume` 升级为可安装的宿主适配器：实现经过调用方身份认证、用户/会话绑定和防重放校验的完成 Webhook；只有可信宿主回调边界验证通过后，才允许把执行授权渲染为 OpenClaw 等聊天渠道中的原生批准/拒绝按钮。当前 URL/Web App 可信页面仍是授权基线，禁止退化为模型可调用的批准命令。

### 2.5 远程代表用户调用

- 当前中心 MCP 使用服务端摘要保存、绑定单一 `userSubject` 的短期 Bearer 身份令牌完成 PoC 引导；它不提供授权码登录、动态客户端注册、刷新令牌或企业身份生命周期，不得直接作为生产 OBO 方案；
- 定义手机/云端 Agent 到 MCP Gateway 的 OBO 委托令牌；
- 令牌绑定发行者、用户、受众、客户端、actor 链、scope、认证强度、防重放标识和有效期；
- 禁止用 service token 加自报 `userId` 的方式代表用户；
- 敏感调用评估 sender-constrained token、设备密钥和 step-up。

### 2.6 跨客户端幂等

去重主域应为：

```text
organization namespace + userSubject
+ capabilityVersion + idempotencyKey
```

默认不包含 clientId/deviceId，避免同一用户从手机和桌面恢复时生成两个写操作。客户端和设备只进入审计链。跨设备继续操作时，按策略重新认证和执行授权。

### 2.7 Worker 租约的真实执行边界

- 当前 PoC 只实现单机、单进程层面的 Profile 租约，能够拒绝同一 Profile 的并发启动；这不是跨主机 fencing，也不能自动证明强制终止后的 Chromium 子进程已经消失；
- 当前 Windows PoC 使用 DPAPI 加密保存进程级浏览器 Session Cookie；Linux/容器和跨主机部署必须改接 Vault/KMS 或等价信封加密，并完成密钥轮换、撤销、备份恢复和内存清理，不能降级为明文 `storage_state.json`；
- 生产 Worker 应使用容器、受限 OS 身份或 Windows Job Object/Linux cgroup 等进程组机制，保证 Worker 退出时浏览器子进程随之终止，并由监督器识别和隔离残留 Profile；
- 遗留网页不理解 fencing token；过期 Worker 仍可能完成已经开始的点击；
- 生产写流量需经过单一会话代理或可撤销网络出口；
- 租约失效时切断 Worker 到目标系统的出站访问；
- 已越过最终提交边界的动作进入 `unknown` 和对账，不能声称可以取消。

### 2.8 原子派发与故障矩阵

- 当前 PoC 已为首个出差申请草稿实现不可变计划、独立可信操作卡、一次性授权、提交边界消费、`unknown` 状态和待发回读；这些是验证样板，不等同于多节点生产原子性；
- 命令与 dispatch-outbox 同事务落库，发布器和周期扫描器负责补发；
- 定义卡住状态、步骤心跳、孤儿任务和超时扫描；
- 为 IdP、策略、Vault/KMS、账本、队列、证据存储、模型、通知、DNS/TLS/NTP 和目标系统建立故障行为矩阵；
- 明确不同风险等级在依赖故障时是拒绝、排队、只读降级还是转人工。

## 3. 企业试点前补齐

### 3.1 生产控制和事件响应

- 建立受审计的运维 CLI/API/控制台；
- 支持按系统、能力、用户、版本暂停，关闭写入但保留只读，冻结/排空队列，隔离会话及切断 Worker 出站；
- 支持按用户冻结认证挑战、撤销中心会话、强制重新登录，并监控异常登录和账号锁定风险；
- 建立 break-glass MFA、双人授权和安全事件响应流程；
- 覆盖批量错误写入、跨用户泄露、凭据泄露、供应链污染和目标系统异常。

### 3.2 SLO、错误预算和人工对账

- 定义受理可用性、排队/启动/完成时延、数据新鲜度、后置条件成功率、`unknown` 比例和最长未决时间；
- 未授权副作用和重复副作用作为零容忍指标；
- 明确人工对账负责人、处理 SLO、最大积压年龄、结案权限和停止写入阈值；
- 告警必须绑定责任人和 Runbook。

### 3.3 灾备和人工业务连续性

- 为命令账本、去重、身份绑定、配置、证据和事件水位定义 RPO/RTO；
- 明确备份、密钥恢复、恢复顺序、旧区域隔离和定期演练；
- AgentBridge 不可用时，用户仍可直接人工使用遗留系统；
- 恢复后核对人工与智能体可能重复执行的操作。

### 3.4 连接器生命周期和升级

- 每个连接器登记业务/技术所有者、系统联系人、关键度、支持级别、兼容矩阵、维护窗口和退役日期；
- 生命周期：`discovery → shadow → canary → production → degraded/quarantined → deprecated → retired`；
- 在途操作固定 Capability、Workflow、Adapter、Skill 和模型策略版本；
- 中心浏览器、驱动、HTTP Adapter 和登录 Adapter 采用受控发布环，兼顾安全补丁和兼容回归；客户端扩展运行时已经删除，不得以调试或迁移名义重新成为代码或生产依赖。

### 3.5 审计、隐私与凭据生命周期

- 审计采用 append-only/WORM、哈希链、可信时间或等价防篡改机制；
- 分别定义模型会话、截图、HAR、下载、推送 Token、操作账本的保留和删除策略；
- 补充数据最小化、法律保全、数据驻留和行业合规评估；
- 完善凭据注册、轮换、KMS/HSM、备份恢复、内存和临时文件清理；
- 针对 Kerberos、NTLM、客户端证书、ActiveX、USB Key 等特殊认证制定专门方案。

## 4. 平台成熟阶段

- 智能体调用预算、循环检测和费用型拒绝服务控制；
- 浏览器进程、损坏 Profile、缓存、下载和临时证据的回收及安全擦除；
- 模型/OCR、浏览器、存储、网络、许可和人工对账的成本核算；
- 国际化、时区、日期/数字格式和移动确认界面可访问性；
- 用户申诉、争议处理和可读回执；
- 主数据映射、账号同名、业务主键变化和事件 Schema 演进；
- SBOM、构建 provenance、制品签名、漏洞扫描及供应链吊销机制。

## 5. 启动条件

出现以下任一情况时，从本 Backlog 提升对应事项：

- 准备接入真实生产数据或高敏数据；
- 准备开放任何 W2/W3 写能力；
- 准备让云端/手机智能体访问企业数据；
- 准备持久保存遗留系统密码、批量启用认证卡片或开放复杂 MFA/桌面接管；
- 用户数、系统数或浏览器 Worker 数需要水平扩展；
- 需要生产 SLA、灾备、审计举证或监管验收；
- 出现共享账号、代办、多个角色或复杂身份生命周期。
